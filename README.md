# PaintAI

PaintAI is a FastAPI + vanilla JavaScript web app for architectural paint color visualization. The current AI workflow sends the uploaded image, project type, and selected paint colors to Gemini image editing (`gemini-2.5-flash-image`) and displays a before/after comparison.

## Current AI Workflow

```text
Frontend: upload image
  -> user chooses interior/exterior
  -> user selects HEX colors for each architectural detail
  -> frontend sends payload to backend
  -> backend converts HEX colors to English architectural color names
  -> backend builds a standardized English architectural prompt
  -> backend sends original image + prompt to Gemini image API
  -> Gemini returns a base64 image
  -> frontend shows before/after comparison slider
```

Example prompt color schedule generated by the backend:

```text
- wall-main (main exterior facade wall): soft warm beige, exact paint HEX #C8B08A
- trim (trim): charcoal black, exact paint HEX #222222
```

## Features

- Upload JPG, PNG, or WEBP images.
- Choose interior or exterior workflow.
- Assign paint colors by architectural part.
- Generate AI paint previews with Gemini image editing.
- Compare original and generated result with a before/after slider.
- Browse paint colors, project templates, saved designs, and catalog data.
- Optional local AI stack for YOLO/SAM-style detection outside serverless deploys.

## Quick Start

Install lightweight runtime dependencies:

```bash
pip install -r requirements.txt
```

Create `.env` in the project root:

```env
GEMINI_API_KEY=your-gemini-api-key
AI_IMAGE_PROVIDER=gemini
GEMINI_IMAGE_MODEL=gemini-2.5-flash-image
DISABLE_LOCAL_AI=1
```

Run locally:

```bash
python server.py
```

Open:

```text
http://127.0.0.1:8000
```

Alternative Uvicorn command:

```bash
python -m uvicorn server:app --host 127.0.0.1 --port 8000
```

## Gemini API Key

Create a key at:

```text
https://aistudio.google.com/app/apikey
```

The UI also has an **API Key** button. A key entered in the UI is stored in browser `localStorage` and is sent as `api_key` with AI requests.

Priority order:

```text
UI API key
  -> GEMINI_API_KEY environment variable
```

If you change `.env`, restart the server. If you change the key in the UI, no restart is needed.

## Important Gemini Quota Note

`gemini-2.5-flash-image` / Gemini image generation may not have enough free quota for real image editing. If the API returns:

```text
429 RESOURCE_EXHAUSTED
Quota exceeded
free_tier_requests, limit: 0
```

then the key/project does not currently have usable image-generation quota. Enable billing or use a project/key with quota:

```text
https://ai.dev/rate-limit
https://ai.google.dev/gemini-api/docs/rate-limits
```

## Generate Colors Request

```json
{
  "image": "data:image/jpeg;base64,...",
  "projectType": "interior",
  "paintAreas": {
    "wall-main": "#C8B08A",
    "accent": "#FFFFFF"
  },
  "detectedAreas": [
    {
      "id": "wall-main",
      "label": "main wall / primary interior wall surface",
      "displayLabel": "Main wall",
      "hex": "#C8B08A"
    }
  ],
  "imageProvider": "gemini",
  "api_key": null
}
```

Supported providers:

```text
local
openai
gpt-image
gemini
nano-banana
```

The frontend currently sends `imageProvider: "gemini"` for AI image generation.

## Environment Variables

```env

AI_IMAGE_PROVIDER=gemini
GEMINI_IMAGE_MODEL=gemini-2.5-flash-image

```

## Vercel Deployment

This repo is configured for Vercel with:

- `api/index.py` as the serverless FastAPI entrypoint.
- `vercel.json` routing `/api/*` to the serverless function and static assets to `/static`.
- `DISABLE_LOCAL_AI=1` so Vercel does not import YOLO/Torch/OpenCV.
- Lightweight `requirements.txt` for serverless builds.

Set these Vercel environment variables:

```env
GEMINI_API_KEY=your-gemini-api-key
AI_IMAGE_PROVIDER=gemini
GEMINI_IMAGE_MODEL=gemini-2.5-flash-image
DISABLE_LOCAL_AI=1
AI_IMAGE_REQUEST_TIMEOUT_SECONDS=90
```

Do not upload `.env`. It is ignored by `.vercelignore`.

Deploy:

```bash
vercel
```

or through the Vercel Git integration.

## Local AI Dependencies

The deployment dependency file intentionally does not include heavy local AI packages. For local YOLO/SAM experiments, install:

```bash
pip install -r requirements-local-ai.txt
```

This may install `torch`, `ultralytics`, and `opencv-python`, which are not suitable for Vercel serverless.

## Troubleshooting

### Gemini key is valid but image generation fails with 429

The key works, but the project has no available quota for the image model. Enable billing or use a different key/project with image-generation quota.

### Unexpected model name format

Make sure:

```env
GEMINI_IMAGE_MODEL=gemini-2.5-flash-image
```

Do not put an API key or full URL in `GEMINI_IMAGE_MODEL`.

### Vercel build fails on Torch/OpenCV

Use the current lightweight `requirements.txt`. Do not put `torch`, `ultralytics`, or `opencv-python` back into `requirements.txt`; keep them in `requirements-local-ai.txt`.

### Image generation times out

Gemini image editing can be slow. Vercel function duration depends on your plan. The current `vercel.json` sets `maxDuration` to 60 seconds.
